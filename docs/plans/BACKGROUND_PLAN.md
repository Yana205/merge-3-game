# Plan: Game Background

**Execution branch:** `feature/game-background` · **Skill:** `/game-background` · **Depends on:** nothing

## Goal

Replace the flat camera-color background with themed art that matches the
pixel-art gem set, without hurting board readability.

## Current state

- The camera clears to a solid dark blue; there is no background object in
  `mainGame.unity`.
- The board sits centered; gems are colorful pixel art (see
  `Assets/_Project/Art/Gems/`).

## Asset options (pick one at execution time)

1. **Canva generation (recommended, consistent with gems)** — generate a
   pixel-art "crystal cave / gem mine" backdrop, muted and dark so the bright
   gems pop. `generate-design` with `phone_wallpaper` (portrait) type, prompt
   for: 16-bit pixel art, dark desaturated purples/blues, no focal objects in
   the center, no text. Export plain PNG (custom-size export fails on this
   plan; default size is fine).
2. **Free packs** — itch.io "puzzle background" free packs or OpenGameArt
   (filter CC0). License check per pack; record in CREDITS.md.
3. **Procedural fallback (zero assets)** — a large radial-gradient sprite
   generated by script (dark edges, slightly lighter center) — cheap, always
   fits, no style risk.

## Integration steps

1. Save art to `Assets/_Project/Art/Backgrounds/` + CREDITS.md entry.
2. Import as Sprite (single), no mipmaps, compression on (it's a big texture).
3. Add a `Background` GameObject to `mainGame.unity` via the Unity MCP
   (never hand-edit the scene): SpriteRenderer, sorting order well behind
   cells/items (check Cell/Item sorting first with `gameobject-component-get`).
4. Scale to fill the camera at any aspect: compute from
   `Camera.orthographicSize` — cover height `2*size` and width
   `2*size*aspect`; scale the sprite up to the larger factor (a small
   `BackgroundFitter` MonoBehaviour in `Scripts/UI/`, or set a generous fixed
   scale if we accept overscan).
5. Readability pass: if gems blend into the art, darken the background sprite
   color (e.g. 70% gray tint) or add a semi-transparent dark panel behind the
   board area.

## Verification

Play mode via Unity MCP; `screenshot-game-view` at 16:9 landscape AND a
portrait resolution; confirm gems/labels clearly readable, no background seams
at the edges.
